# MOB-010: Internationalization (Hebrew + Arabic + English)

**Priority:** P0 — all UI text depends on i18n
**Blocked by:** MOB-001 (project scaffold, font config, locale setup)
**Estimated effort:** 4 days
**Contract:** `contracts/mobile/lib/core/config/app_config.dart` (AppLocales, TypographyTokens)

---

## Context
Cena supports three locales: Hebrew (primary, RTL), Arabic (RTL, for Israeli Arab students ~30% of the population + future MENA expansion), and English (LTR fallback). Both Hebrew and Arabic are right-to-left, which affects layout mirroring, text alignment, icon positioning, and scrolling direction. The Flutter `intl` package with ARB files provides compile-time-safe localization. Font switching is locale-aware: Heebo for Hebrew, Noto Sans Arabic for Arabic, Inter for English. Device locale detection determines the default, with manual override in settings. The l10n pipeline uses `flutter_gen` to produce `AppLocalizations` from ARB files.

### MOB-010.2: Locale-Aware Font Switching
**Files:**
- `lib/core/theme/locale_font_resolver.dart`
- `lib/core/theme/cena_theme.dart` (update from MOB-001)

**Acceptance:**
- [ ] `LocaleFontResolver` class (or extension on `TypographyTokens`):
  - `resolve(Locale locale)` returns the primary `TextTheme` with correct font family
  - Hebrew locale -> all text styles use `Heebo` font family
  - Arabic locale -> all text styles use `Noto Sans Arabic` font family
  - English locale -> all text styles use `Inter` font family
- [ ] `CenaTheme.light(Locale locale)` and `CenaTheme.dark(Locale locale)` build `ThemeData` with locale-resolved font
- [ ] Font fallback chain: primary locale font -> system font (never shows tofu/boxes)
- [ ] Math/code content always uses `JetBrains Mono` regardless of locale (`TypographyTokens.monoFontFamily`)
- [ ] `TextTheme` sizes match `TypographyTokens`: `displayLarge=32`, `displayMedium=28`, `headlineLarge=24`, `headlineMedium=20`, `titleLarge=18`, `titleMedium=16`, `bodyLarge=16`, `bodyMedium=14`, `bodySmall=12`, `labelLarge=14`, `labelMedium=12`, `labelSmall=10`
- [ ] Font switching is reactive: when user changes locale in settings, theme rebuilds immediately

**Test:**
```dart
test('LocaleFontResolver returns Heebo for Hebrew', () {

  expect(textTheme.headlineLarge?.fontFamily, equals('Heebo'));
```

- [ ] `DirectionalScaffold` wrapper: applies `Directionality` based on current locale, flips all layout for RTL

### MOB-010.4: Locale Detection & Manual Override
**Files:**
- `lib/core/services/locale_service.dart`
- `lib/features/settings/locale_picker.dart`

**Acceptance:**
- [ ] `LocaleService` class:
  - `detectDeviceLocale()`: reads `PlatformDispatcher.instance.locale`, maps to nearest supported locale
  - `getSavedLocale()`: reads user's manual override from `SharedPreferences` (or Hive)
  - `saveLocale(Locale)`: persists locale preference
  - `getEffectiveLocale()`: returns saved locale if set, otherwise device locale, with fallback to Hebrew
- [ ] `LocalePicker` widget in settings:
  - Shows 3 options: "עברית" (Hebrew), "العربية" (Arabic), "English"
  - Each option shows language name in its own script
  - Selected locale highlighted
  - On selection: saves via `LocaleService`, rebuilds app theme immediately
  - "Auto (device language)" option at top that clears the override
- [ ] On first launch: detect device locale, if Arabic -> use Arabic, if Hebrew -> use Hebrew, else -> Hebrew
- [ ] Locale change is immediate (no restart required) — uses a `localeProvider` in Riverpod

**Test:**

## Rollback Criteria
- If `flutter gen-l10n` has bugs with complex ICU plural rules in Arabic: simplify to basic string interpolation and revisit plurals later
- If Noto Sans Arabic has rendering issues on older Android (API 23): bundle a subset font or fall back to system Arabic font
- If RTL layout causes regressions in third-party widgets: wrap problematic widgets in explicit `Directionality(textDirection: TextDirection.ltr)` overrides
- If locale detection is unreliable: default to Hebrew for all Israeli devices and show locale picker on first launch
