## Context

CNT-003 built the expert review UI for batch-generated questions (accept/edit/reject). This task extends it to handle ingested and re-created questions from CNT-008, adds bias auditing, auto-approval for high-confidence items, moderation analytics, and a structured approval workflow with escalation paths.

Moderators are licensed Bagrut teachers. They validate mathematical correctness, pedagogical quality, language accuracy, and cultural appropriateness. Their decisions determine what students see.

### CNT-009.3: Auto-Approval Engine

**Files to create/modify:**
- `src/Cena.Ingest/AutoApproval/AutoApprovalEngine.cs` — rule-based auto-approval
- `config/ingestion/auto_approval.yaml` — thresholds, spot-check percentage

**Acceptance:**
- [ ] Auto-approve items meeting ALL criteria:
  - `quality_scores.overall >= 0.95`
  - `quality_scores.math_correctness == 1.0`
  - `source.type == 'bagrut_exam'` (from official published exam)
  - `dedup_hashes.semantic_embedding` max similarity to existing < 0.5
  - `classification.bloom_confidence >= 0.90`
- [ ] Auto-approved items: `status: 'auto_approved'`, bypass review queue
- [ ] Spot-check: random 10% of auto-approved items ALSO sent to review queue with `priority: 'spot_check'`
- [ ] If spot-check rejection rate > 5%, auto-approval is paused for that concept cluster and all auto-approved items are re-queued for human review
- [ ] Dashboard metric: auto-approval rate, spot-check rejection rate, false positive rate
- [ ] Thresholds configurable per concept cluster (some concepts may need stricter review)

**Test:**
```python
def test_auto_approval_all_criteria_met():
    item = make_item(
        overall=0.97, math_correctness=1.0,
        source_type="bagrut_exam", max_similarity=0.3, bloom_confidence=0.95
    )
    assert auto_approve(item) is True

def test_auto_approval_fails_low_math():
    item = make_item(overall=0.97, math_correctness=0.0)
    assert auto_approve(item) is False

def test_spot_check_pauses_on_high_rejection():
    auto_approve_items(100, concept="derivatives")
    reject_spot_checks(6, concept="derivatives")  # 6% > 5% threshold
    assert auto_approval_enabled("derivatives") is False
```

### CNT-009.4: Bias Audit Checklist

**Files to create/modify:**
- `src/Cena.Ingest/Classify/BiasAuditor.cs` — automated bias detection
- `config/ingestion/bias_checklist.yaml` — bias categories and rules
- Extends CNT-003 review UI: add bias audit panel to review screen

**Acceptance:**
- [ ] Automated bias checks (LLM-based) run during classification (CNT-008.5):
  - **Representational**: Names in word problems balanced across gender, ethnicity, socioeconomic contexts
  - **Cultural**: Scenarios appropriate for both Hebrew-sector and Arab-sector students
  - **Gender in Arabic**: Verify grammatical gender agreement doesn't create unintended bias
  - **Stereotyping**: Flag questions that use stereotyped professional/social roles
  - **Accessibility**: Flag questions that assume visual/physical ability unnecessarily
- [ ] Each bias check produces: `{ category, detected: bool, confidence, explanation }`
- [ ] Items with any bias flag are routed to human review (cannot be auto-approved)
- [ ] Moderator UI shows bias flags with explanations on the review screen
- [ ] Moderator can: confirm bias (reject item), dismiss flag (false positive), or edit to remove bias
- [ ] Bias metrics tracked: flags per concept, false positive rate, most common bias categories

**Test:**
```python
def test_bias_detects_gendered_name():
    item = make_item(stem_he="דוד קנה 5 תפוחים וגם 3 אגסים")  # "David bought..."
    # Not flagged — using a name is fine
    assert bias_audit(item).detected is False

def test_bias_flags_stereotype():
    item = make_item(stem_he="האם שלחה את הילדים לחנות")  # "The mother sent the children..."
    # Flagged — assumes mother does shopping (gender stereotype)
    result = bias_audit(item)
    assert result.detected is True
    assert result.category == "stereotyping"
```

- `src/Cena.Web/Controllers/ModerationAnalyticsController.cs` — dashboard endpoints
- Extends `config/content/escalation-thresholds.yaml`

**Acceptance:**
- [ ] Track rejection rate per: concept cluster, question_type, source_type, generation_model, moderator
- [ ] `GET /api/admin/moderation/analytics` — dashboard data:
  - Overall approval/rejection/edit rates
  - Rejection rate by concept (heatmap)
  - Rejection rate by reason (pie chart)
  - Moderator throughput (items/hour)
  - Auto-approval rate and spot-check results
  - Queue depth over time
- [ ] Escalation triggers:
  - 20-40% rejection per concept → alert Content Admin
  - \> 40% rejection per concept → pause generation, escalate to Curriculum Lead
  - \> 3 consecutive rejections of same item type → flag prompt template issue
  - Moderator disagrees with auto-approval spot-check → tighten auto-approval thresholds
- [ ] Weekly digest email to Content Admin: top rejection reasons, worst-performing concepts, moderator stats

**Test:**
```csharp
[Fact]

    }
    var alerts = await _analytics.GetAlerts("derivatives");
```
