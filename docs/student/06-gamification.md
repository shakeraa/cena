# 06 — Gamification

## Overview

Gamification layers motivation on top of the learning loop without gaming the learning itself. Matches the mobile spec exactly (so XP, streaks, and badges are consistent across surfaces) and adds a few web-only affordances.

The gamification system must obey the **age-aware variable rewards** rules from `variable_rewards.dart`: for students 18+, XP and streaks are de-emphasized in favor of mastery goals.

## Mechanics

### XP & Levels

- Awarded on: correct answer, first-try-correct bonus, streak milestones, quest completion, badge unlock.
- Level curve matches mobile: level `n` requires `n * 100 + (n-1)^2 * 25` XP (confirm in `gamification_widgets.dart`).
- Level-up triggers `<CelebrationOverlay>` + dedicated Rive animation.

### Streaks

- Calendar-day streak — at least one session of ≥ 5 minutes per day.
- Streak freeze items (earned or purchased with XP) grant one-day grace.
- Streak milestones: 3 / 7 / 14 / 30 / 60 / 100 / 365 days, each unlocks a badge.
- Streak broken → compassionate recovery message, no punishment animation.

### Badges

- Categories: **engagement**, **mastery**, **social**, **exploration**, **perseverance**, **creativity**.
- 3D badge widget renders with a WebGL/Rive preview (matches `badge_3d_widget.dart`).
- Badge detail dialog shows: how to earn, rarity, unlock date, shareable link.
- Locked badges are silhouetted with a cryptic hint.

### Quests (Daily / Weekly)

- Generated by `quest_generator.dart` equivalent on the server.
- Quest types: complete N sessions, master N concepts, hit M% accuracy in a subject, maintain streak, teach back N times, help N peers.
- Progress is realtime (SignalR events).
- Completion triggers XP + bonus and pushes a quest-complete celebration.

### Leaderboards

- Scopes: **class**, **friends**, **global**, **regional** (new — web-only toggle), **all-time** vs **weekly**.
- Ranking metric: weekly XP by default; configurable to mastery points.
- Own rank always visible regardless of scroll position (sticky row).

### Momentum Meter

- Visualizes recent activity intensity over the last 7 days.
- Matches mobile `momentum_meter.dart` — horizontal bar chart with ambient color from flow state.

### Variable Rewards (Ethical Persuasion)

- Rewards are not purely random; they follow the research in `ethical-persuasion-research.md` and `variable_rewards.dart`.
- Over age 18, XP multipliers and surprise rewards are opt-in.
- Under age 18, the system favors intrinsic rewards (progress, mastery) and caps extrinsic intensity.

## Pages

### `/progress` → Gamification Section

The main `/progress` page has a gamification tab / section that aggregates:
- XP + level + progress ring
- Streak widget (current, longest, next milestone)
- Badge showcase (carousel of last 5 unlocked + "View all")
- Momentum meter
- Active quests

### `/progress/badges`

Full badge gallery with filter chips per category. Clicking a badge opens `<BadgeDetailDialog>`.

### `/social/leaderboard`

See [11-social-learning.md](11-social-learning.md) — the leaderboard UI is shared between gamification and social.

### Celebration Overlays

`<CelebrationOverlay>` is a global singleton that can be triggered from anywhere in the app via a `useCelebration()` composable. Shows:
- Confetti / particles (Rive)
- Headline ("Level up!", "Badge unlocked!", "7-day streak!")
- Subcopy
- Dismiss or "Share" CTA
- Auto-dismiss after 4 s

## Web-Specific Enhancements

- **Badge wall on profile** — large grid of all badges, filterable, sortable, with rarity indicators.
- **Shareable badge cards** — generate an Open Graph image with the badge + student name (no PII beyond display name).
- **Leaderboard embeds** — a teacher can embed a read-only class leaderboard into an LMS via `?embed=1`.
- **Quest journal** — a dedicated page with past quests, completion rates, and average time-to-complete.
- **Celebration sound mix** — subtle audio toggle per celebration type in settings.
- **Desktop notifications** — opt-in browser notifications for quest completion, badge unlock, and streak save reminders (browser Notification API).
- **Seasonal events** — themed skins, quests, and badges tied to holidays or school calendar; configured server-side, rendered with time-limited themes.
