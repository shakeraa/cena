# GD-007: PhET-Style Student Interview Protocol

> **Goal**: Iterative user testing for sandbox physics features
> **Based on**: PhET student-interview methodology (Finkelstein et al. 2005)

## Protocol

### Session setup (5 min)
- Student uses Cena on their own device (phone or tablet)
- Interviewer observes, does not instruct
- Screen recording + think-aloud protocol

### Exploration phase (10 min)
- Student opens sandbox physics (inclined plane / free-body diagram)
- No instruction — observe what they try first
- Note: which controls do they find? Which confuse them?

### Guided task (10 min)
- "Can you set the angle to 30 degrees and find the net force?"
- Observe: do they understand the constraint (one-knob pattern)?
- Note: where do they get stuck? What language do they use?

### Reflection (5 min)
- "What was confusing?" / "What did you expect to happen?"
- "Would you use this instead of drawing on paper?"

### Data collection
- Think-aloud transcript
- Screen recording with timestamps
- Interviewer field notes (confusion points, delight points, bugs)
- Post-session survey (3 questions, Likert scale)

## Sample size
- 5 students per iteration (Nielsen's usability testing threshold)
- 3 iterations minimum before launch
- Recruit from: Cena pilot schools (Nazareth, Umm al-Fahm, Rahat per GD-008)

## Analysis
- Affinity diagram of confusion points → feature backlog items
- Compare: does sandbox outperform paper-and-pencil on conceptual items?
